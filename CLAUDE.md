> 이 문서는 '산업안전보건 논문 경진대회' 출품작의 핵심 가설을 데이터로 증명하기 위한
> **작업 지침(instruction)** 이다. Claude는 이 프로젝트의 모든 작업에서 아래 규칙을 따른다.

---

## 0. 프로젝트 한 줄 요약

> **기업이 정부 규제·과징금을 피하려 "서류상 안전(형식)"에는 투자하지만 "실제로 사고를 줄이는 안전(실질)"은 방치한다 — 이 괴리를 `SDI = Substantive − Formal`로 수치화한다.**

SDI가 **음수(−)** 일수록 "형식만 있고 실질이 부족한 안전 형식주의(safety decoupling)" 사업장이고,
SDI가 **양수(+)** 일수록 "실질이 형식을 앞서는 진짜 안전" 사업장이다.

## 1. 핵심 원칙 (항상 지킬 것)

1. **데이터가 먼저, 추측은 금지.** 변수 분류·계산·해석은 반드시 실제 코드북과 raw data 값에 근거한다. 변수 이름만 보고 추측하지 않는다. 불확실하면 `value_counts()`·코드북을 먼저 확인한다.
2. **코드북은 진실의 원천.** 모든 변수의 의미·척도 방향·결측코드는 `__제10차_산업안전보건_실태조사_코드북_230824.xlsx`를 근거로 한다.
3. **업종은 신중히 다룬다.** 건설/제조/서비스는 **설문 문항 번호와 구성이 서로 다르고, 변수명이 같아도 의미가 다르다.** 업종별 개별 분석(Track A)과 통합 분석(Track B)을 모두 수행하되, 통합 시에는 변수명이 아니라 **의미 기반 개념 매핑(§3-1)** 으로만 정렬한다 (§3 참조).
4. **재현 가능성.** 모든 분석은 Jupyter Notebook에서 셀 단위로 실행 가능해야 하며, 난수가 개입하면 `random_state` / seed를 고정한다.
5. **결측·무응답 처리 명시.** `9`, `99`, `999`, `99999` 등은 실제 값이 아니라 무응답 코드다. 분석 전 반드시 `NaN` 처리한다 (§4).
6. **방향성 정렬(reverse coding) 필수.** 변수마다 "높을수록 안전한가"가 다르다. SDI 계산 전 모든 변수를 **"값↑ = 안전↑"** 방향으로 통일한다 (§5.3).
7. **가중치 인식.** 데이터에는 표준화가중치 `WT2`가 있다. 기술통계·집계 시 가중치 적용 여부를 항상 명시하고, EFA는 비가중으로 수행하되 SDI 집계 해석 시 가중 결과를 병기한다.
8. **결과는 항상 해석과 함께.** 숫자만 출력하지 말고, 그 숫자가 가설(형식주의)에 대해 무엇을 의미하는지 한국어로 해석을 덧붙인다.
9. **변수명 존재 검증 필수.** 매핑표·코드북에서 참조한 변수명을 코드에 쓰기 전에 반드시 실제 데이터프레임에 존재하는지 확인한다. 확인 방법: `sorted(df.filter(like='Q16').columns.tolist())`. 코드북 표기(예: `Q16_1_1`)와 매핑표 표기(예: `Q16_1`)가 다를 수 있으므로 실데이터 열 이름을 직접 출력해 대조한다. 존재하지 않는 변수명을 그대로 쓰면 KeyError 없이 NaN 컬럼이 생성되어 분석이 무결없이 오염된다.
10. **F/S 변수 수 균형 명시.** Formal 변수와 Substantive 변수의 개수가 현저히 다르면(예: F=4개, S=13개) SDI 범위·분산이 S 쪽으로 왜곡될 수 있다. 변수 목록을 확정할 때 F/S 각각의 개수를 명시하고, 불균형이 크면 그 이유를 보고서에 기재한다. 사용자가 직접 F/S 변수를 지정한 경우에는 그 목록을 코드 상단 주석에 원본 코드명으로 그대로 옮겨 적는다.
11. **보고서·테이블에 영어 alias 금지.** 분석 과정에서 내부 처리 편의를 위해 영어 label(예: `mgmt_emphasis`, `rule_have`)을 쓰더라도, 최종 출력 테이블·보고서·주석에는 반드시 **원본 변수 코드**(예: `Q17_1`, `Q17_11`)를 사용한다. 변환이 있으면 "Q17_11: min-max 정규화(원값 1~5 → 0~1)" 형식으로 원 코드+변환 내용을 병기한다. Claude가 임의로 붙인 영어 이름을 연구자가 추적할 수 없으면 재현성이 깨진다.

### ⚠️ 절대 함정: 같은 변수명 ≠ 같은 의미 (실측 검증됨)

세 raw 파일을 단순 비교하면 변수명이 82개 겹친다. **그러나 변수명이 같아도 의미는 거의 모두 다르다.** 변수명으로 `merge`/`concat` 하면 분석이 통째로 무너진다. 실측 예시:

| 변수명 | 제조·서비스업 의미 | 건설업 의미 |
|---|---|---|
| `Q17_1` | 경영진이 안전을 강조한다 | 작업반장이 역할·책임을 안다 |
| `Q14_1` | 소음으로 인한 위험 | 위험성평가 안 한 이유 |
| `Q13_1` | KOSHA-MS 인증 | 소음으로 인한 위험 |
| `Q19_1` | 산업안전보건 교육·훈련 | 일반건강진단 실시 |
| `Q11` | 안전보건관리담당자 선임 | 관리감독자 수 |

→ 우연히 의미가 일치하는 건 `ID`, `Q5`(노조), `Q27_*`(사고), `WT2`, `code2`, `SQ3` 정도뿐이다.
**통합 분석은 반드시 §3-1의 "의미 기반 개념 매핑"을 사용한다. 변수명 교집합을 쓰지 않는다.**

### 3-2. 분석 전략: 개별 + 통합 비교 (Two-track)

본 연구는 **두 트랙을 모두 수행하고 비교**한다.

- **Track A — 업종별 개별 분석:** 각 업종 전용 변수 전체로 EFA→분류→SDI를 따로 산출. 업종 특성(건설=관리비 항목, 서비스=감정노동 등)을 살린 정밀 분석. 업종 간 SDI 절대수준은 직접 비교하지 않고 "업종 내 분포·순위"로 해석.
- **Track B — 통합 분석:** §3-1 개념 매핑으로 세 업종 공통 17문항(필요시 확장)을 정렬해 `industry` 라벨을 붙여 하나의 데이터프레임으로 통합. 동일 척도로 SDI를 산출하여 **업종 간 직접 비교**가 가능. 통합 EFA로 형식/실질 2요인이 업종 무관하게 재현되는지 확인(구성타당도).
- **비교 산출물:** ① 업종별 SDI 분포 비교(박스플롯), ② 통합 SDI의 업종별 평균 차이검정(ANOVA/Kruskal-Wallis), ③ Track A·B 결과의 일관성 점검.

**원칙: 업종별 코드북 시트를 각각 읽어 전용 변수 리스트를 만든다(Track A). 통합(Track B)은 변수명이 아니라 §3-1 개념 매핑으로 정렬한 뒤에만 합친다.**

## 4. 결측·무응답 코드 (분석 전 NaN 처리)

| 척도 유형 | 정상 값 | 무응답 코드 → NaN |
|---|---|---|
| 5점 리커트(그렇다/위험/심각) | 1~5 | `9` |
| 4점 리커트(노력 정도 등) | 1~4 | `9` |
| 이분형(예/아니요) | 1, 2 | `9` |
| 위원회·인지 등 3~4점 | 1~3(4) | `9` |
| 인원수·사고건수(문자열코딩) | 0 이상 정수 | `999`, `99999` |
| 비율(%) | 0~100 | `999` |

처리 예: `df[likert_cols] = df[likert_cols].replace(9, np.nan)` / `df[count_cols] = df[count_cols].replace([999, 99999], np.nan)`
코드값은 변수마다 다를 수 있으니 **일괄 처리 전 코드북으로 검증**한다.

## 5. 5단계 워크플로우

### 1단계 — 연구 설계
- 목적: 형식적 안전관리(Formal)와 실질적 안전관리(Substantive)를 데이터로 구분하여 **SDI 지수**를 구축.
- 접근: **탐색적 요인분석(EFA) + 코드북 기반 해석**을 결합한 데이터 기반 분류.
- 산출물: 업종별 SDI 점수 + 업종 비교 + (선택) SDI와 사고율(Q27)의 관계 검증.

### 2단계 — 데이터 준비
1. 업종별 CSV를 각각 로드 (인코딩 UTF-8).
2. 코드북에서 해당 업종 변수 사전(dict: 변수명→설명, 코드값)을 파싱.
3. 무응답 코드 NaN 처리 (§4).
4. 분석 대상 변수만 선별: **태도·행동·제도 척도형 문항 위주**, 단순 식별·인구통계(ID, SQ*, 지역, 종사자수 등)와 결과변수(Q27 사고)는 EFA에서 제외하고 별도 보관.

### 3단계 — 탐색적 요인분석(EFA)
- **추출법:** Principal Axis Factoring (주축 분해 — 공통요인 추출).
- **회전:** Varimax (직교회전 — 한 요인에 변수가 강하게 적재되도록).
- **요인 수:** 2개 가정 (형식 vs 실질). 단, Kaiser 기준(고유값>1)·스크리도표·평행분석으로 2요인 적합성을 **반드시 검토**하고 결과를 보고한다.
- **사전 점검:** KMO 표본적합도(>0.6 권장), Bartlett 구형성 검정(p<0.05).
- **권장 라이브러리:** `factor_analyzer` (`FactorAnalyzer(n_factors=2, method='principal', rotation='varimax')`).
- EFA는 척도가 동일한 문항군(예: `Q17_*`/`Q16_*` 17~18문항 리커트) 안에서 수행하는 것이 안정적이다. 척도가 섞이면 표준화 후 투입하거나 문항군별로 나눠 실행한다.
- 결측은 EFA 전에 listwise 또는 문항평균 대치 중 택일하고 방식을 명시한다.

### 4단계 — 변수 분류 (핵심 단계)
**EFA 적재량 + 코드북 의미**를 결합해 각 변수를 4범주로 분류한다.

| 범주 | 정의 | 키워드 신호 |
|---|---|---|
| **Formal (형식)** | 제도·규정·인증·서류·선임·문서 존재 여부 → "안전이 시스템으로 존재" | 인증(KOSHA-MS, ISO45001), 전담부서 유무, 선임신고, 위원회 구성, 규정·절차 보유, 교육 실시 여부, 위험성평가 문서작업 시행 여부 |
| **Substantive (실질)** | 행동·참여·실효성·개선효과 → "안전이 실제로 작동" | "실질적으로 도움", "자발적으로 개선", 근로자 참여, 위험성 낮추려는 노력, 사후관리 이행 확인, 작업거부권, 위원회 의결사항 지속관리 |
| **Review (검토)** | 형식·실질 양면 가능 — 수동 판단 필요 | 교육 방법, 관리감독자 역량, 협력·원청 소통 등 |
| **Exclude (제외)** | SDI와 무관 (식별·인구통계·결과변수) | ID, SQ*, 지역, 종사자 수, 사고건수(Q27), 가중치 |

- **분류 방법:** ① 키워드 기반 자동 1차 분류 → ② EFA 적재 패턴으로 검증 → ③ 코드북 문구로 수동 보정.
- **결정적 단서(코드북 실측):** `Q17_11`("체계적인 안전 규정과 절차를 **가지고 있다**")=형식 신호, `Q17_12`("안전 규정과 절차는 사고 예방에 **실질적으로 도움**이 된다")=실질 신호. 이런 **형식–실질 쌍(pair)** 이 SDI의 핵심 증거다. 우선적으로 식별할 것.
- 분류 결과는 `변수명 / 설명 / 범주 / 근거(EFA적재·키워드)` 표로 산출한다.

### 5단계 — SDI 지수 구성
1. **방향 정렬:** 모든 채택 변수를 "값↑ = 안전 양호" 방향으로 통일 (역문항은 `6 - x` 등으로 reverse). 위험요인 인식(Q14/Q13), 스트레스 심각도 등은 의미를 따져 신중히 처리.
2. **정규화:** 각 변수를 0~1 Min-Max 또는 z-score로 표준화 (방법 명시·일관 적용).
3. **요인 점수화:**
   - `Formal Factor(F)` = 형식 변수들의 (표준화) 합성 점수.
   - `Substantive Factor(S)` = 실질 변수들의 (표준화) 합성 점수.
   - 단순평균 또는 EFA 요인점수(factor score) 중 택일하고 근거를 적는다.
   - **변수 목록은 반드시 원본 코드명으로 명시한다.** 예: `formal_vars = ['Q17_7', 'Q17_9', 'Q17_11', 'Q17_13']` — 영어 alias 불가.
   - **F/S 변수 수를 확인한다.** 개수를 출력(`print(f"F 변수 수: {len(formal_vars)}, S 변수 수: {len(subst_vars)}")`)하고, 비율이 1:3 초과면 불균형 원인과 그 영향을 보고서에 기재한다.
4. **SDI 계산:** `SDI = Substantive − Formal` (사업장 단위).
5. **해석:**
   - `SDI > 0` : 실질 > 형식 → **진짜 안전**.
   - `SDI < 0` : 형식 > 실질 → **형식주의(decoupling)**.
   - `SDI ≈ 0` : 형식·실질 균형.
6. **(권장) 가설 검증:** SDI(또는 Formal/Substantive)와 결과변수 `Q27`(사고·질병 건수, 사고율로 환산)의 상관·회귀를 본다. **형식점수만 높고 SDI가 낮은 집단에서 사고가 더 많다면** 가설이 데이터로 지지된다.

formal_vars = [...]      # 분류 확정 후 채움

### 6-1. Track B — 3업종 통합 분석 (검증 완료)

> 변수명이 아니라 **§3-1 개념 매핑**으로 정렬해 합친다. 아래 코드는 실제로 실행해 7,308개 사업장 통합·2요인 재현·업종별 SDI 비교까지 확인됨.

```python
import pandas as pd, numpy as np
from factor_analyzer import FactorAnalyzer
from scipy import stats
BASE = "/mnt/project/"

# 공통라벨 -> (제조·서비스 변수, 건설 변수)  ※ §3-1 검증 매핑
MAP = {
 'mgmt_emphasis':('Q17_1','Q16_1'), 'mgmt_priority':('Q17_2','Q16_2'),
 'mgmt_value':('Q17_3','Q16_3'),    'meeting':('Q17_4','Q16_4'),
 'open_talk':('Q17_5','Q16_5'),     'ask_opinion':('Q17_6','Q16_6'),
 'suggest_sys':('Q17_7','Q16_7'),   'suggest_resp':('Q17_8','Q16_8'),
 'edu_provide':('Q17_9','Q16_9'),   'edu_effect':('Q17_10','Q16_10'),
 'rule_have':('Q17_11','Q16_11'),   'rule_effect':('Q17_12','Q16_12'),
 'facility':('Q17_13','Q16_13'),    'tidy':('Q17_14','Q16_14'),
 'std_follow':('Q17_15','Q16_16'),  'refuse':('Q17_16','Q16_17'),
 'voluntary':('Q17_17','Q16_18'),
}
def load_mapped(fname, kind):           # kind: 'ms'(제조/서비스) | 'con'(건설)
    df = pd.read_csv(BASE+fname)
    out = pd.DataFrame()
    for label,(msv,conv) in MAP.items():
        col = msv if kind=='ms' else conv
        out[label] = df[col].replace(9, np.nan)   # 무응답 NaN
    out['WT2'] = df['WT2']
    return out

mfg = load_mapped("제10차_산업안전보건_실태조사_raw_data_제조업_230824.csv","ms");  mfg['industry']='제조'
svc = load_mapped("제10차_산업안전보건_실태조사_raw_data_서비스업_230824.csv","ms"); svc['industry']='서비스'
con = load_mapped("제10차_산업안전보건_실태조사_raw_data_건설업_230824.CSV","con");  con['industry']='건설'
alld = pd.concat([mfg,svc,con], ignore_index=True)   # → (7308, 19)

items = list(MAP.keys())

# 통합 EFA: 형식/실질 2요인이 업종 무관하게 재현되는지(구성타당도)
fa = FactorAnalyzer(n_factors=2, method="principal", rotation="varimax")
fa.fit(alld[items].dropna())
print("통합 EFA 고유값:", np.round(fa.get_eigenvalues()[0][:4],2))  # [9.76 1.16 ...] → 2요인

# 형식/실질 합성 → 통합 SDI (분류는 §3-1 성격열 기반, EFA로 보정)
formal = ['suggest_sys','edu_provide','rule_have','facility']
subst  = [c for c in items if c not in formal]
mm = lambda s:(s-s.min())/(s.max()-s.min())
alld['F'] = alld[formal].apply(mm).mean(axis=1)
alld['S'] = alld[subst ].apply(mm).mean(axis=1)
alld['SDI'] = alld['S'] - alld['F']

# 업종 간 직접 비교 (통합의 핵심 가치)
print(alld.groupby('industry')['SDI'].agg(['mean','median','std']).round(4))
g = [alld.loc[alld.industry==k,'SDI'].dropna() for k in ['제조','서비스','건설']]
print("Kruskal-Wallis:", stats.kruskal(*g))   # 업종 차이 유의성
```
