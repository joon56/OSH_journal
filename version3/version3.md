## 1. 변경된 연구 가설

### 주 가설
> "SDI가 낮은(형식 > 실질) 사업장일수록 위험성평가를 형식적으로만 수행한다."

- `β_SDI > 0` (SDI 높을수록 위험성평가 실질성 점수 높음) → 가설 지지
- 즉, 실질 안전이 형식을 앞서는 사업장에서 위험성평가도 더 실질적으로 수행됨

### 보조 가설
> "SDI가 낮은 사업장에서 작업환경측정 및 건강진단 실시율도 낮다."

- 예방활동 실시 여부에서도 같은 패턴이 확인되면 가설의 외적 타당도 강화

### 참고 비교
> "Q27 사고율과의 관계는 Version 2 결과와 비교·병기한다." (부록)

## 2. 횡단 설계 정당화 논리 (전략 B — 역인과 차단)

**역인과 시나리오**: "위험성평가 수준이 낮아서 SDI가 낮아진다"

**반박 근거:**
1. SDI의 F 변수(인증·선임·전담부서·위원회)는 위험성평가 실시 수준과 무관한 제도 변수다. 위험성평가 수준이 F_score를 결정하지 않는다.
2. SDI의 S 변수(경영진 강조·교육 실효성·행동 준수·감독자 역량)는 위험성평가 실시 여부와 별개로 측정된다.
3. 따라서 위험성평가 수준이 SDI를 유발한다(역인과)는 경로가 구조적으로 차단된다.
4. 논리 방향: **낮은 안전 문화(저SDI) → 예방 활동의 형식화** 가 더 타당하다.

**추가 서술 (논문용):**
"본 연구는 인과 관계가 아닌 '구조적 연관성(structural association)'을 검증한다. SDI(안전관리 문화의 형식-실질 배분)와 위험성평가 수준(가장 핵심적 예방 활동)이 같은 방향으로 움직인다면, 이는 형식주의적 안전 관리가 예방 활동 전반에 걸쳐 나타나는 패턴임을 시사한다."

## 6. 분석 설계

### 6-1. 건설업

**주 모형 — OLS (연속형 ra_score):**
```
ra_score_con ~ SDI + elder_rate + foreign_rate + female_rate + log(Q4_1)
```

**주 모형 — 순서형 로지스틱 (4범주 ra_score):**
```
OrderedLogit(ra_score_con, X_con)
```

**보조 모형 — 이진 로지스틱:**
```
작업환경측정_실시 ~ SDI + 통제변수
건강진단_실시    ~ SDI + 통제변수
```

### 6-2. 제조+서비스업

**주 모형:**
```
ra_score_ms ~ SDI + ind_mfg(업종더미) + elder_rate + foreign_rate + female_rate + log(Q1_1)
```

**보조 모형:**
```
작업환경측정_실시 ~ SDI + ind_mfg + 통제변수
건강진단_실시    ~ SDI + ind_mfg + 통제변수
```

### 6-3. 가설 지지 기준

| 모형 | 가설 지지 조건 |
|---|---|
| OLS (ra_score) | β_SDI > 0, p < 0.05 |
| 순서형 로지스틱 | β_SDI > 0, p < 0.05 |
| 보조 이진 로지스틱 | β_SDI > 0, p < 0.1 (보조이므로 기준 완화) |

> OLS와 순서형 로지스틱 방향이 일치하면 강한 증거.

## 8. 단계별 작업 순서

```
Step 1: 변수 검증 (§4 스크립트 실행)
         → Q14/Q15, Q21_1/Q22_1, Q19_1/Q20_1 실데이터 확인

Step 2: 데이터 로드 + 전처리
         → Version 2 무응답 처리 동일 적용
         → ra_score 생성 (위험성평가 재코딩)
         → 예방활동 이진 변수 생성

Step 3: SDI 계산 (Version 2 동일)
         → F_score, S_score, SDI 산출
         → 기술통계 출력

Step 4: 결과변수 기술통계
         → ra_score 분포 (업종별)
         → 예방활동 실시율 (업종별)
         → SDI 분위별 ra_score 평균 비교 (시각화)

Step 5: 회귀 분석
         → 건설업: OLS + 순서형 로지스틱 + 보조 이진
         → 제조+서비스: 동일
         → β_SDI 부호·유의성·OLS vs 순서형 방향 일치 확인

Step 6: 역인과 논리 검토
         → SDI와 ra_score의 이변량 산점도·상관 출력
         → "SDI가 먼저 결정되고 ra_score는 그 표현"임을 논리적으로 서술

Step 7: 보고서 생성
         → output/v3/SDI_report_v3.md
         → output/v3/ 산출물 저장
```
