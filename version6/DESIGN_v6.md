### 1-1. 코딩 오류

### 1-2. DV 선택 문제

| 버전 | DV | 문제점 |
|------|-----|--------|
| V1~V2 | Q27 사고 건수 | 극단적 IRR, 역인과(사고→F 투자), 과소보고 |
| V3~V5 | Q15/Q14 (ra_score) | **빈도 측정** (미실시/비정기/정기). 실질성(quality) 아님. F_score가 구조적으로 RA를 더 잘 예측 → 가설과 반대 방향 필연적 |

### 1-3. ra_score를 DV로 쓰면 왜 가설이 불지지되는가

```
F_score 구성 요소:
- KOSHA-MS 인증 → 인증 조건에 RA 정기 실시 의무
- ISO 45001     → 동일
- 전담부서(Q6)  → 부서가 RA를 기획·실행
- 안전관리자    → 핵심 업무가 RA

∴ F_score ↑ → RA 빈도(ra_score) ↑ (구조적 필연)

S_score 구성 요소:
- 스트레스 관리 (Q16_2_*)
- 경영진 안전 강조 (Q17_1~3)
- 교육 효과 (Q17_9~10)
- 근로자 행동 (Q17_15~17)

∴ S_score와 RA 빈도는 간접 관계 → β_S < β_F 필연적

결과: SDI = S - F 가 높을수록 RA 빈도 낮아짐 (β_SDI < 0)
→ 가설과 반대
```

**DV(RA 빈도)와 가설(RA 실질성) 간의 개념 불일치가 근본 원인.**

### 1-4. RA 실질성 측정 후보 변수들의 문제점

| 변수 | 내용 | 문제 |
|------|------|------|
| Q15_2_2 / Q14_2_2 | 위험성평가 시 근로자 참여 여부 | 이진. 93% "예" — 분산 없음 |
| Q15_2_3 / Q14_2_3 | 위험성 감소를 위한 노력 | 5점 리커트. 87.5%가 4~5점 — 천장 효과 |
| Q15_2_1 / Q14_2_1 | RA 수행 주체 (내부/외부/혼합) | 명목 변수. 형식성 간접 측정 가능하나 약함 |

**결론: 이 설문에는 RA 실질성을 직접 측정하며 충분한 분산을 가진 DV 변수가 존재하지 않는다.**

## 2. Version 6 핵심 전환

> **DV를 RA 빈도(ra_score)에서 실제 안전 결과(Q27 사고 발생)로 전환한다.**

### 이유

SDI 이론의 궁극적 예측은 단순하다:
- 형식 중심(SDI 낮음) → 서류는 갖추지만 실제 사고는 줄지 않음
- 실질 중심(SDI 높음) → 실제 안전문화 → 사고 감소

Q27(업무상 사고)은 SDI 이론이 예측해야 하는 **최종 결과변수(terminal outcome)**다. RA 빈도는 "중간 과정 변수"에 불과하다.

### V1의 실패(Q27 극단적 IRR)를 반복하지 않는 방법

V1에서 Q27을 카운트 변수로 직접 사용하면서 극단적 계수가 나왔다. V6에서는:
- **이진화(binary)**: 1년 내 사고 발생 여부 (0/1)
- **로지스틱 회귀**: 계수 안정적, 해석 명확

## 3. Version 6 사전 확정 결정사항 (Pre-Analysis Plan)

### 결정 1: 결과변수 (DV)

**주 DV: `any_accident` — 2021년 업무상 사고 발생 여부**

```python
# Q27_3_1: 2021년 업무상 사고, 요양 4~89일
# Q27_3_2: 2021년 업무상 사고, 요양 90일 이상
# Q27_3_3: 2021년 업무상 사고, 사망
# 무응답 코드 999 → NaN 처리 후
any_accident = ((Q27_3_1 + Q27_3_2 + Q27_3_3) > 0).astype(int)
# 0 = 사고 없음, 1 = 사고 발생
```

- 2021년 데이터 사용 이유: 2021년이 설문 시점에 가장 근접한 최근 연도
- 사망 포함 이유: 사망은 가장 심각한 사고 결과이며 SDI 이론의 핵심 타겟
- 질병(Q27_4_*)은 제외: 사고(accident)와 직업병(disease)은 발생 메커니즘이 다름 — 보조 분석에서만 검토

**보조 DV: `accident_rate` — 사고 발생률 (로그 변환)**

```python
total_accidents = Q27_3_1 + Q27_3_2 + Q27_3_3  # NaN 처리 후
accident_rate = total_accidents / workers  # 100인당 사고율
log_accident_rate = np.log1p(accident_rate)  # 0포함 로그 처리
```

### 결정 2: 가설 방향 (사전 확정)

| 수준 | 조건 | 검증 방법 | 방향 |
|------|------|----------|------|
| 수준 1 | β_SDI < 0, p < 0.05 | 모형 A: any_accident ~ SDI + controls | SDI↑ → 사고 확률↓ |
| 수준 2 | β_S < β_F, Wald p < 0.05 | 모형 B: any_accident ~ F_score + S_score | ΔP(사고)/ΔS < ΔP(사고)/ΔF |

**이론적 근거:**
- 형식 준수 중심(F↑, S 상대적 낮음) → RA 서류는 갖추지만 실제 위험 감소 미흡 → 사고↑
- 실질 안전문화 중심(S↑) → 경영진 의지·관리감독자 역량·근로자 참여 → 실제 위험 감소 → 사고↓
- 따라서 β_SDI < 0, |β_S| > |β_F| → β_S < β_F (둘 다 음수일 때)

| 변수 | 정의 | 코드 (제조·서비스) | 코드 (건설) |
|------|------|-----------------|-----------|
| log_workers | 종사자 수 로그 (사고 노출량) | ln(Q1_1+1) | ln(Q4_1+1) |
| ind_mfg | 제조업 더미 | industry == '제조' | — |
| elder_rate | 50세 이상 비율 | Q1_2/Q1_1 | Q4_2/Q4_1 |

| female_rate | 여성 비율 | Q1_4/Q1_1 | Q4_4/Q4_1 |

> **노출(exposure) 처리**: 사고 건수 DV에서는 log_workers를 offset으로 처리하는 것이 원칙이나, binary DV(any_accident)에서는 통제변수로 투입. 회사 규모가 클수록 절대 사고 수가 많아지는 효과를 통제.

### 결정 5: 역인과 처리 방침 (사전 선언)

역인과 가능성: 사고를 많이 경험한 기업이 이후 F_score를 높임(규제 준수 강화)

**처리 방법**:
1. **이론적 논거**: SDI는 사업장의 구조적 특성(인증 보유, 안전문화 수준)을 측정하므로 단기 사고에 의한 역인과보다 장기 문화적 특성이 더 안정적으로 반영됨
2. **방향성 약화 논거**: 역인과가 있다면 오히려 F_score가 상향 편향되어 β_SDI의 크기를 과소 추정(보수적 추정). 결과가 유의하면 역인과를 감안해도 효과 존재
3. **한계 명시**: 패널 자료 없이 인과관계를 확정할 수 없음을 투명하게 기술

### 결정 6: 회귀 모형

**모형 A (주 가설, 수준 1):**
```
logit(P(any_accident)) ~ β_SDI·SDI + β₂·log_workers + β₃·ind_mfg 
                        + β₄·elder_rate + β₅·foreign_rate + β₆·female_rate
```
H₁: β_SDI < 0, p < 0.05

**모형 B (F/S 분리, 수준 2):**
```
logit(P(any_accident)) ~ β_F·F_score + β_S·S_score + (통제변수)
Wald: H₀: β_S = β_F  →  H₁: β_S < β_F (단측, 더 음수)
```

**보조 OLS (해석 편의):**
```
any_accident ~ SDI + (통제변수)  [선형 확률 모형, LPM]
```

### 결정 7: 강건성 검증 계획

| 검증 | 방법 | 판단 기준 |
|------|------|----------|
| Q17_13 민감도 | Q17_13을 S로 이동 후 재실행 | β_SDI 부호(음) 유지 여부 |
| 50인↑ 제한 | 중소기업 편향 제거 | 방향 일치 |
| OLS vs 로지스틱 | LPM vs 로지스틱 | 방향 일치 |
| 2021년 → 2020년 | Q27_1_* 사용 | 방향 일치 |
| 사고+질병 복합 | Q27_3+Q27_4 포함 | 방향 일치 |

### 결정 8: 보조 분석 — RA 빈도 결과 보고 방식

V5(수정 후)의 RA 빈도 결과(β_F > β_S)는 **별도 보조 결과**로 보고:

> "형식 준수 중심 사업장이 위험성평가를 더 자주 실시한다는 사실은 SDI 이론의 또 다른 측면을 지지한다: 형식 인프라가 compliance 행동을 유도하지만, 이것이 실제 안전 결과(사고 감소)로 이어지는지는 별개의 문제다(§4-3). 주 분석(§4-2)에서 SDI가 사고 발생을 음으로 예측하는 것은 형식적 RA 실시와 실제 안전 결과 사이의 괴리를 직접 시사한다."

## 4. 논문 서술 전략

### 4-1. 연구 기여의 3층 구조

```
층 1 (방법론): SDI 지수 구성
  - 기존 연구: 단일 지표(사고율, 인증 여부)
  - 본 연구: 형식/실질 이차원 분해 → 괴리 자체를 측정

층 2 (실증): SDI → 사고 발생 예측
  - β_SDI < 0: 형식주의 기업에서 사고 더 많이 발생
  - β_S < β_F: 실질 문화가 형식 인프라보다 사고 예방에 효과적

층 3 (이론): 경계 조건 탐색
  - 건설업 이질성: 내부 SDI vs 외부 거버넌스
  - 조절효과로 산업별 메커니즘 차이 확인
```

### 4-2. 데이터 한계 → 연구 기여로의 전환

| 데이터 한계 | 논문 기술 방식 |
|------------|-------------|
| 역인과 가능성 | "본 연구는 선행 연구들과 동일하게 횡단면 자료를 사용하며, SDI의 구조적 특성으로 인해 역인과 효과가 제한될 것으로 판단하나 패널 자료를 통한 확인이 필요하다. 이 한계에도 불구하고 본 연구는 SDI 지수의 이론적 구성과 초기 타당성 검증이라는 방법론적 기여에 초점을 맞춘다." |
| Q27 과소보고 가능성 | "응답자가 사고 건수를 실제보다 낮게 보고하는 경향이 있다(사회적 바람직성 편향). 이 편향이 SDI 수준과 독립적으로 발생한다면 결과의 방향성에 영향을 미치지 않는다." |
| RA 실질성 직접 측정 불가 | "본 자료는 RA 실시 빈도를 측정하며 실질성을 직접 측정하지 않는다. 그러나 보조 분석에서 형식 중심 사업장이 RA를 더 자주 실시하면서도 사고율이 더 높다는 결과는, 형식적 RA 실시가 실제 사고 예방으로 이어지지 않음을 간접적으로 시사한다." |

### 4-3. 최종 논문 결과 서술 시나리오 (목표)

```
"제조·서비스업(n=5,320)에서 SDI는 업무상 사고 발생 확률을 유의하게 
음으로 예측하였다(β < 0, p < 0.05). F_score·S_score 분리 투입 
모형에서 |β_S| > |β_F|로 실질 안전문화가 형식 인프라보다 사고 
예방에 더 강하게 기여함이 확인되었다. 

보조 분석에서 형식 인프라(F_score)가 위험성평가 실시 빈도를 더 
강하게 예측하는 반면, 실질 문화(S_score)가 실제 사고 발생을 더 
강하게 억제하는 결과는 안전 형식주의의 핵심 명제 — '서류상 
안전이 실제 안전을 보장하지 않는다' — 를 직접 데이터로 
뒷받침한다."
```
